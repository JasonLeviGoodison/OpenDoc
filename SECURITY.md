# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Send reports to:

- `jasongoodison1234@gmail.com`

Include:

- A clear description of the issue
- Reproduction steps
- Impact assessment
- Any suggested fix or mitigation

## Response Expectations

- Initial acknowledgement target: 5 business days
- Status update target: 10 business days

## Scope

Security reports are especially helpful for:

- Authentication or authorization bypasses
- Document access control issues
- Token forgery or session leakage
- Sensitive data exposure
- Injection vulnerabilities
- Supply chain or dependency issues with practical impact
